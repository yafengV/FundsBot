# FundsBot PRD 10：迭代计划与功能状态看板

> 目标：提供“按功能迭代”的单页视图，确保每个功能都具备可开发、可测试、可验收的交付条件。

## 1. 状态定义（统一）
- `draft`：需求草案，结构未冻结
- `review`：评审中（产品/研发/测试）
- `ready`：可进入开发（AC/TC/接口/DoD齐全）
- `dev`：研发中
- `test`：测试中
- `done`：完成并验收
- `blocked`：有阻塞

## 2. 迭代规划（建议）

### Iteration-1（MVP闭环）
- F01 账本管理
- F02 持仓管理
- F04 实时估值与首页总览

**目标**：用户完成手动录入后，可看到“今天赚亏 + 总资产”

### Iteration-2（导入能力）
- F03 持仓导入（CSV+OCR）

**目标**：降低首次录入成本，提升导入完成率

### Iteration-3（日结与提醒）
- F05 净值更新与提醒

**目标**：晚间净值更新后可触达并给出最终收益结论

### Iteration-4（分析与治理）
- F06 报表分析（周报月报）
- F07 数据口径与对账
- F08 系统设置与非功能要求

**目标**：形成可复盘、可监控、可治理的完整产品能力

## 4. Ready Gate（进入开发前必须满足）
每个功能在状态从 `review` → `ready` 前，必须满足：
1. 用户故事（US）完整且有验收标准（AC）
2. 功能需求（FR）可实现，无歧义
3. 接口契约已明确（字段、错误码、边界）
4. 数据口径明确（输入/计算/输出）
5. 测试用例（TC）齐全并映射 AC
6. 研发任务拆解（BE/FE/QA/OPS）+ DoD 已定义

## 5. 周例会更新规则（建议）
- 更新频率：每周一次
- 更新人：产品负责人（主）、研发/测试负责人（协）
- 更新内容：状态、阻塞项、风险、下周目标
