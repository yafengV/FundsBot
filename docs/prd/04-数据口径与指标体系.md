# FundsBot PRD 04：数据口径与指标体系

> 文档目标：统一“怎么算”，避免前后端、产品、运营对数据理解不一致。

## 1. 指标口径定义

## 1.1 总资产
- 定义：所有持有中基金的最新估值市值之和
- 公式：`总资产 = Σ(份额 × 最新估值净值)`

## 1.2 当日预估盈亏
- 定义：以估值净值计算的当日盈亏（盘中动态）
- 说明：非最终结算值，晚间以净值结算替换

## 1.3 当日最终盈亏
- 定义：净值更新后，以实际净值计算的当日盈亏
- 用途：通知、日报、周报月报入库值

## 1.4 累计盈亏
- 定义：当前市值 + 累计已实现收益 - 累计投入本金
- 说明：与交易记录强相关，清仓后仍保留历史贡献

## 1.5 收益率
- 定义：`收益率 = 累计盈亏 / 累计投入本金`
- 异常：投入本金为 0 时返回 --

## 2. 数据刷新与时效
- 交易时段：估值数据分钟级刷新（来源：tushare fund_quote）
- 非交易时段：展示最近一次有效值
- 净值更新：按基金公司公布时间异步更新（来源：tushare fund_nav）

## 2.1 数据源说明
- **实时估值**: 通过 tushare `fund_quote()` 接口获取
- **历史净值**: 通过 tushare `fund_nav()` 接口获取  
- **基金基础信息**: 通过 tushare `fund_basic()` 接口获取
- **数据缓存**: 建议实现本地缓存减少API调用频率

## 3. 数据分层建议
1. 原始层：行情源/导入原文
2. 标准层：统一字段、统一币种
3. 指标层：收益计算与报表快照
4. 展示层：首页/列表/报表结果

## 4. 报表快照策略
- 周报、月报按生成时刻落快照
- 快照不可被后续行情变动覆盖
- 用户可重算时需保留版本记录

## 5. 数据质量规则
- 导入重复检测：基金代码 + 账本 + 买入日期 + 成本区间
- 异常值报警：单日收益波动超过阈值触发校验
- 丢数监控：关键指标缺失率 > 1% 告警

## 6. 埋点指标体系（运营）
- 漏斗：注册 -> 导入 -> 首页查看 -> 开启提醒 -> 看周报
- 行为：日打开次数、首页停留时长、通知点击率
- 质量：导入错误率、提醒失败率、数据延迟率

## 7. 隐私与合规
- 用户可发起“删除全部数据”
- 报表分享默认脱敏（不带账户标识）
- 不向第三方暴露用户可识别投资数据
