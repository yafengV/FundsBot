# FundsBot PRD 07：测试用例与验收矩阵（QA执行版）

> 目标：测试可直接按该文档编写测试计划并执行。

## 1. 测试范围
- 功能：P0 全覆盖
- 接口：关键读写接口全覆盖
- 数据：收益口径一致性
- 非功能：性能、稳定性、可观测性

## 2. 环境与前置
- 环境：dev/staging/prod
- 前置账户：
  - U1（单账本）
  - U2（多账本）
  - U3（空数据）
- 基金样例：稳定基金、波动基金、停牌/异常基金

## 3. 用例矩阵（核心）

## 3.1 账本模块
- TC-LED-001 创建账本成功
- TC-LED-002 重名账本创建失败（FND-1003）
- TC-LED-003 汇总视图资产正确

## 3.2 持仓模块
- TC-POS-001 手动添加成功
- TC-POS-002 基金代码不存在失败（FND-1001）
- TC-POS-003 增仓后加权成本正确
- TC-POS-004 减仓超过可用份额失败（FND-1003）
- TC-POS-005 清仓后状态变更正确
- TC-POS-006 删除持仓需确认 Token

## 3.3 导入模块
- TC-IMP-001 CSV 全量成功
- TC-IMP-002 CSV 部分失败并返回行级错误
- TC-IMP-003 OCR 草稿含低置信字段标红
- TC-IMP-004 OCR 未确认不能入库

## 3.4 首页与估值
- TC-DSH-001 首页总资产计算正确
- TC-DSH-002 当日预估盈亏随行情刷新变化
- TC-DSH-003 行情超时显示延迟提示

## 3.5 净值提醒
- TC-NOT-001 净值更新后 10 分钟内触达
- TC-NOT-002 免打扰生效（仅站内）
- TC-NOT-003 同类阈值提醒每日不超过 1 次

## 3.6 报表
- TC-RPT-001 周报指标完整
- TC-RPT-002 月报快照不可被后续行情改写
- TC-RPT-003 多账本切换结果正确

## 4. 数据正确性对账用例（必须）
- TC-DATA-001 固定样本对账（人工基准表）
- TC-DATA-002 边界值（0份额、0成本、极大份额）
- TC-DATA-003 跨日切换口径（预估 -> 最终）

## 5. 性能与稳定性
- TC-PERF-001 首页接口 p95 < 500ms
- TC-PERF-002 持仓列表 500 条加载 < 1.5s
- TC-STB-001 连续 24h 任务稳定，无 P0 告警

## 6. 验收门槛（发布阻断）
- P0 用例通过率 100%
- P1 用例通过率 >= 95%
- 数据对账误差 <= 0.1%
- Sev1/Sev2 缺陷 = 0
- 崩溃率/错误率在基线以内

## 7. 缺陷分级
- Sev1：收益计算错误、数据串账、无法进入首页
- Sev2：核心流程失败（导入/提醒/报表）
- Sev3：非核心功能异常
- Sev4：UI 文案/轻微体验问题
