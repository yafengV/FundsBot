# FundsBot PRD 01：模块拆解与需求清单

> 文档目标：将产品能力拆成清晰模块，输出优先级明确、可排期的需求清单。

## 1. 模块地图（v1.0）
1. 账户与账本模块
2. 持仓导入与维护模块
3. 实时估值与收益总览模块
4. 净值更新与通知模块
5. 分析与报表模块
6. 系统设置与偏好模块

## 2. 需求清单（按优先级）

## 2.1 P0（MVP 必须）
### A. 账户与账本
- A1：支持创建多个账本（默认账本 + 自定义）
- A2：账本切换后数据隔离
- A3：提供“全部账本汇总”视图

### B. 持仓导入与维护
- B1：手动添加持仓（基金代码、份额、成本、平台、时间）
- B2：编辑持仓（增减仓、成本重算）
- B3：删除持仓（需二次确认）
- B4：CSV 导入模板与校验
- B5：截图 OCR 导入“草稿列表”+ 用户确认

### C. 实时估值总览
- C1：首页显示总资产、累计盈亏、当日预估盈亏
- C2：持仓列表显示基金涨跌、预估净值、浮动收益
- C3：支持排序（涨跌幅/收益额/收益率）
- C4：支持按账本/平台筛选

### D. 净值提醒
- D1：净值更新完成提醒（全组合）
- D2：收益阈值提醒（例如亏损超 X 元）
- D3：免打扰时段设置

### E. 报表
- E1：周报（收益、波动、贡献Top）
- E2：月报（收益趋势、账户对比、基金表现）

## 2.2 P1（v1.1 推荐）
- P1-1：基金标签体系（高风险/低波动/行业）
- P1-2：基金对比页（最多 3 只）
- P1-3：可导出 PDF 报表
- P1-4：家庭成员共享只读视图

## 2.3 P2（后续探索）
- P2-1：组合健康评分
- P2-2：智能复盘建议
- P2-3：策略回放（仅分析，不交易）

## 3. 不同角色的需求映射
- 新手用户：C1/C2/D1（看结果）
- 进阶用户：B2/C3/E1/E2（看结构）
- 重度用户：A1/A3/B4/B5（看全局）

## 4. 需求依赖关系
- B（导入）是 C/D/E 的前置
- C（总览）是 D（提醒阈值）与 E（报表）的数据前置
- A（账本）影响 B/C/E 的数据隔离和汇总口径
