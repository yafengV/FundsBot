# FundsBot PRD 00：总览与产品定义

> 文档目标：定义产品边界、目标用户、核心价值与版本范围。该文档是后续所有 PRD 子文档的基础。

## 1. 文档信息
- 产品：FundsBot
- 版本：v1.0（MVP）
- 更新时间：2026-03-07
- 对标产品：养基宝（功能体验对标，不涉及品牌和实现细节复制）
- 文档状态：草案 -> 可进入评审

## 2. 背景与问题
基金用户通常在多个平台持有基金，常见问题：
1. **数据分散**：无法快速看到“全局总资产/总收益”
2. **查看成本高**：白天频繁切换 APP 查看涨跌
3. **晚间反馈延迟**：净值更新后不能第一时间得到收益结论
4. **复盘能力弱**：缺乏周/月层面的结构化分析

## 3. 产品愿景与定位
### 3.1 愿景
让普通投资者“低成本、可持续”地管理基金投资过程，降低信息摩擦和决策焦虑。

### 3.2 定位
**跨平台基金持仓管理与分析助手（不提供交易）**。

### 3.3 核心价值主张
- 一站式持仓视图
- 白天看估值，晚上看净值
- 自动提醒 + 报表复盘
- 多账户管理能力

## 4. 用户与场景
## 4.1 核心用户画像
1. **轻度用户（新手）**
   - 关注“今天赚了/亏了多少”
   - 只需要简单、清晰的总览
2. **中度用户（持续定投）**
   - 关注收益率、成本、回撤
   - 需要基金维度分析与时间趋势
3. **重度用户（多平台多账户）**
   - 关注账户分组管理和汇总
   - 需要更稳定的数据同步和报表能力

## 4.2 高频使用场景
- 早盘/午盘/收盘查看组合实时浮动
- 晚间净值更新后接收最终收益通知
- 周末查看周报，月初查看月报
- 新增买入后快速更新持仓成本

## 5. 产品边界
## 5.1 In Scope（v1.0）
- 持仓录入（手动、CSV、截图 OCR 导入草稿）
- 实时估值与收益看板
- 晚间净值更新提醒
- 盈亏分析（基金维度、账户维度）
- 周报/月报
- 多账户账本

## 5.2 Out of Scope（v1.0）
- 下单交易、申购赎回执行
- 券商/股票交易全能力
- 智能投顾与自动调仓
- 高频策略回测

## 6. 产品原则
1. **先准确，后丰富**：核心指标可核对，优先于炫酷图表
2. **先高频，后深度**：先满足“今天赚亏”和“净值通知”
3. **先可解释，后智能化**：所有收益结果可追溯来源
4. **默认安全**：导入与提醒不泄露敏感信息

## 7. 版本递进策略
- **阶段 A：可用（MVP）**
  - 完成持仓管理、收益总览、净值通知
- **阶段 B：好用（V1）**
  - 完成周/月报、多账户汇总、数据质量提升
- **阶段 C：耐用（V1.1+）**
  - 个性化提醒、高级分析、跨端同步与导出增强

## 8. 关键成功指标（产品层）
- 首次导入完成率 >= 85%
- 日活用户中“查看首页总览”渗透率 >= 70%
- 净值通知触达率 >= 98%
- 周报阅读率 >= 30%
- 用户 30 日留存（有持仓用户） >= 15%
