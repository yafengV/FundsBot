# FundsBot PRD 02：用户故事与关键流程

> 文档目标：把需求转成可执行用户流程，明确关键页面、状态和异常路径。

## 1. 用户故事（User Stories）

## 1.1 持仓导入
- 作为一个在多个平台持仓的用户，我希望一次性导入所有基金持仓，这样我能快速看到总体收益。
- 验收标准：首次导入路径 <= 3 分钟，导入成功后首页可见总览。

## 1.2 日常查看
- 作为每日会看收益的用户，我希望打开应用就看到今日预估盈亏和涨跌分布，这样我不用逐个平台查看。
- 验收标准：首页关键指标首屏可见，支持一键切换筛选。

## 1.3 晚间提醒
- 作为关注净值更新的用户，我希望净值更新后收到最终收益提醒，这样我可以快速复盘当天表现。
- 验收标准：更新后 10 分钟内推送，点击通知进入当日总结页面。

## 1.4 周月复盘
- 作为定投用户，我希望看到周报和月报，了解盈利来源和拖累项。
- 验收标准：周报/月报均能显示收益、贡献榜、账户对比。

## 2. 核心流程设计

## 2.1 首次使用流程（Onboarding）
1. 进入应用 -> 选择“创建账本”
2. 选择导入方式（手动/CSV/OCR）
3. 完成导入校验 -> 进入首页总览
4. 引导开启净值提醒

失败分支：
- OCR 识别失败 -> 提示手动修正
- 导入字段缺失 -> 指引补全

## 2.2 每日查看流程
1. 首页查看总览
2. 进入持仓列表查看涨跌与收益
3. 点击基金查看详情趋势
4. 返回首页看组合变化

## 2.3 净值提醒流程
1. 系统检测净值更新
2. 计算当日最终收益
3. 触发通知（受免打扰策略影响）
4. 用户点击后进入“今日总结页”

## 2.4 周报/月报流程
1. 系统按周期生成报表快照
2. 用户在报表页查看并筛选账本
3. 用户可分享截图

## 3. 页面与状态清单
- 页面：启动页、首页、持仓页、基金详情页、提醒设置页、报表页、账本管理页
- 状态：首次空状态、导入中、导入失败、数据更新中、无可用净值

## 4. 异常与兜底策略
1. 行情服务异常：展示上次更新时间 + “数据可能延迟”
2. 通知发送失败：站内消息兜底
3. 用户关闭通知权限：在首页显著提示开启
4. OCR 低置信结果：强制人工确认

## 5. 可用性要求
- 核心路径（导入->查看->提醒）不超过 4 次关键点击
- 关键指标字体在移动端首屏可读
- 误操作有撤销或确认机制
