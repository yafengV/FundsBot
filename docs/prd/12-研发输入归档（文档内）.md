# FundsBot PRD 12：研发输入归档（文档内）

> 目标：在“纯PRD阶段”将原本分散在脚本/配置/DDL/OpenAPI中的研发输入统一收敛到文档体系，避免关键信息落在文档之外。

## 1. 环境配置基线（原 .env.example）
- Runtime: `ENV=dev`, `TZ=Asia/Shanghai`, `PORT=8080`
- Database: `DB_URL=postgresql://fundsbot:fundsbot@localhost:5432/fundsbot`
- Cache: `REDIS_URL=redis://localhost:6379/0`
- Security: `JWT_SECRET=<required>`
- Third-party:
  - `QUOTE_API_BASE_URL`, `QUOTE_API_KEY`
  - `NAV_API_BASE_URL`, `NAV_API_KEY`
  - `OCR_API_BASE_URL`, `OCR_API_KEY`
- Push/Alert:
  - `FCM_SERVER_KEY`
  - `APNS_KEY_ID`, `APNS_TEAM_ID`, `APNS_BUNDLE_ID`, `APNS_P8_BASE64`
  - `SENTRY_DSN`

## 2. 本地运行编排（原 docker-compose.dev.yml）
- PostgreSQL 16
  - DB: `fundsbot`
  - User/Password: `fundsbot/fundsbot`
  - Port: `5432`
- Redis 7
  - Port: `6379`
  - AOF 开启

## 3. 研发命令约定（原 Makefile）
- `deps`：安装本地依赖（jq/yq/openapi/spectral）
- `up/down/logs`：启动/停止/查看本地依赖服务
- `db-init/db-seed/db-reset`：初始化、灌种子、重建数据库
- `ci`：执行本地CI检查

> PRD阶段不保留可执行脚本，以上仅作为后续研发落地约定。

## 4. API 契约归档（原 OpenAPI）
服务前缀：`/api/v1`

核心资源：
- Ledgers：`GET/POST /ledgers`
- Positions：`GET/POST /positions`，`PATCH/DELETE /positions/{id}`
- Imports：CSV validate/commit，OCR draft/confirm
- Dashboard：`GET /dashboard/overview`
- Quotes：`GET /quotes/realtime`
- Notify：`PUT /notify/rules`，`GET /nav/daily-summary`
- Reports：`GET /reports/weekly`，`GET /reports/monthly`

错误码主线：
- 业务校验：`FND-1001/1002/1003`
- 数据源/行情：`FND-2001/2002/2003/2004/2005`
- 推送：`FND-3001`
- 系统：`FND-9001`

## 5. 数据模型归档（原 DDL）
关键枚举：
- `platform_type`: alipay/tencent/tiantian/xueqiu/other
- `position_status`: holding/cleared/deleted
- `trade_op`: buy/sell/adjust
- `report_type`: weekly/monthly

核心表：
- 用户与账本：`app_user`, `ledger`
- 主数据与持仓：`fund`, `position`, `position_txn`
- 行情与净值：`fund_quote_realtime`, `fund_nav_daily`
- 快照：`pnl_daily_snapshot`, `report_snapshot`
- 提醒与幂等：`notify_rule`, `idempotency_key`

## 6. 联调样本归档（原 seed.sql）
- 用户：1个 demo_user
- 账本：主账户 + 家庭账户
- 基金：3只样本（161725/000001/110022）
- 持仓：3条holding样本
- 净值：同一日样本净值
- 提醒规则：默认开启净值提醒 + 阈值

## 7. CI 检查约定归档（原 ci-check.sh / bootstrap.sh）
- 必要产物检查：API契约、数据模型、样本数据
- OpenAPI lint（spectral）
- SQL可用性检查（本地psql）
- 工具依赖：jq/yq/openapi-generator/spectral
