# F05 功能PRD：净值更新与提醒

## 1. 背景与目标
- 背景问题：用户很难在净值出齐后第一时间知道“当日最终结果”。
- 本功能目标：在净值更新后触达用户，并支持盈亏阈值提醒与免打扰。
- 业务价值：增强留存与复访。

## 2. 范围
### 2.1 In Scope
- 净值更新完成提醒
- 盈利/亏损阈值提醒
- 免打扰时段
- 点击跳转当日总结

### 2.2 Out of Scope
- 个性化文案A/B自动优化
- 多渠道编排策略引擎（v1.1+）

## 3. 用户故事（US）
### US-001 日终关注用户
**Description**：我希望净值更新后收到最终盈亏提醒。

**Acceptance Criteria**：
- AC-001：净值更新后10分钟内完成触达。
- AC-002：点击提醒可跳转当日总结页。

### US-002 风险敏感用户
**Description**：我希望亏损/盈利超过阈值时收到提醒，但不被骚扰。

**Acceptance Criteria**：
- AC-003：同类阈值提醒每日最多1次。
- AC-004：免打扰时段不发外部推送（仅站内）。

## 4. 功能需求（FR）
- FR-001：支持设置提醒规则（阈值、免打扰）。
- FR-002：净值日结完成后触发提醒任务。
- FR-003：支持提醒防抖与去重。

## 5. 交互与流程
- 主流程：设置规则 → 净值更新 → 计算触发条件 → 发送提醒。
- 异常流程：推送失败/重复触发/免打扰拦截。
- 状态流转：`eligible` → `triggered` → `sent/failed/suppressed`。

## 6. 接口契约（功能视角）
- `PUT /api/v1/notify/rules`
- `GET /api/v1/nav/daily-summary`

错误码：
- FND-2002 净值数据缺失
- FND-3001 推送发送失败

## 7. 数据口径与规则
- 触发窗口默认：20:30-23:30。
- 防抖规则：同类阈值提醒每天最多1次。
- 免打扰：仅站内提醒，不发外部通道。

## 8. 异常与降级
- 推送失败：重试3次指数退避。
- 净值缺失：延后补发任务并记录原因。
- 降级：外部推送失败时保底站内消息。

## 9. 埋点与可观测
- 埋点：`notify_rule_update`、`notify_trigger`、`notify_click`。
- 监控：触达率、重复提醒率、失败率。
- 告警：触达率<98% 或失败率>2%。

## 10. 测试用例（TC）
- TC-NOT-001（AC-001）：净值更新后10分钟内触达。
- TC-NOT-002（AC-002）：点击跳转正确。
- TC-NOT-003（AC-003）：防抖生效。
- TC-NOT-004（AC-004）：免打扰生效。

## 11. 研发任务拆解与 DoD
### BE
- 任务：规则存储、触发任务、去重防抖、重试机制。
- DoD：防抖与免打扰场景通过。

### FE
- 任务：提醒规则设置页、当日总结跳转。
- DoD：配置修改实时生效。

### QA
- 任务：时窗、防抖、失败重试、免打扰测试。
- DoD：核心路径100%通过。

### OPS
- 任务：推送成功率监控、失败告警。
- DoD：告警可触发且可恢复。

## 12. 验收标准
- 提醒触达率 >= 98%
- 重复提醒率 < 1%
- 净值提醒时延 <= 10分钟

## 13. 风险与待确认
- 风险：第三方推送通道波动影响触达。
- Open Question：工作日与周末提醒窗口是否不同。
