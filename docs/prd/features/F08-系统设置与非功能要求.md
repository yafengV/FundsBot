# F08 功能PRD：系统设置与非功能要求

## 1. 背景与目标
- 背景问题：缺少统一配置和稳定性保障会导致线上不可控。
- 本功能目标：提供系统配置能力并明确性能、安全、稳定性门槛。
- 业务价值：保障可上线、可运维、可回滚。

## 2. 范围
### 2.1 In Scope
- 通知权限与免打扰配置
- 时区配置
- 数据源配置管理（tushare）
- 日志追踪与告警

### 2.2 Out of Scope
- 多云多活容灾体系（v1.1+）

## 3. 用户故事（US）
### US-001 配置管理员
**Description**：我希望区分测试/生产环境配置，避免误用。

**Acceptance Criteria**：
- AC-001：支持测试与生产环境配置切换。
- AC-002：敏感token仅通过环境变量管理。

### US-002 运维与质量角色
**Description**：我希望系统具备性能、可用性和告警能力。

**Acceptance Criteria**：
- AC-003：首页接口P95 < 500ms。
- AC-004：关键异常可追踪requestId并告警。

## 4. 功能需求（FR）
- FR-001：支持通知、时区、免打扰配置。
- FR-002：支持数据源配置与健康检查。
- FR-003：支持日志追踪与监控告警。

## 5. 交互与流程
- 主流程：修改配置 → 校验 → 生效 → 监控确认。
- 异常流程：配置错误/数据源不可用/性能退化。
- 状态流转：`draft` → `validated` → `active` → `rollback`。

## 6. 接口契约（功能视角）
- `PUT /api/v1/notify/rules`
- 配置中心内部接口（后端）

错误码：
- FND-2003 数据源配置错误（token无效）
- FND-2004 数据源积分不足
- FND-2005 数据源API限流

## 8. 异常与降级
- token无效：立即切换降级模式（使用缓存）。
- 限流/积分不足：降频+缓存+告警。
- 性能退化：触发灰度回滚。

## 9. 埋点与可观测
- 埋点：`config_update`、`datasource_switch`、`rollback_trigger`。
- 监控：API成功率、错误率、积分消耗速率、首页P95。
- 告警：成功率<99.5% 或错误率>1%。

## 10. 测试用例（TC）
- TC-SYS-001（AC-001）：测试/生产配置切换正常。
- TC-SYS-002（AC-002）：token安全检查通过。
- TC-SYS-003（AC-003）：性能压测达标。
- TC-SYS-004（AC-004）：日志与告警链路完整。
- TC-SYS-005：数据源异常降级处理正确。

## 11. 研发任务拆解与 DoD
### BE
- 任务：配置校验、数据源适配、降级逻辑、日志追踪。
- DoD：配置切换与降级用例通过。

### FE
- 任务：配置页、状态反馈、错误提示。
- DoD：配置修改后可见生效状态。

### QA
- 任务：配置切换、性能、异常链路测试。
- DoD：阻断项全部通过。

### OPS
- 任务：监控告警、灰度与回滚预案。
- DoD：演练通过并有记录。

## 12. 验收标准
- 核心可用性 >= 99.9%
- 首页接口P95 < 500ms
- 数据源调用成功率 >= 99.5%
- Sev1/Sev2 缺陷为0

## 13. 风险与待确认
- 风险：单一数据源故障影响范围大。
- Open Question：是否在v1.0引入备选数据源。
