# F07 功能PRD：数据口径与对账

## 1. 背景与目标
- 背景问题：看板、提醒、报表口径不一致会直接损害信任。
- 本功能目标：建立统一口径、对账机制与异常告警。
- 业务价值：保障“数据可信”。

## 2. 范围
### 2.1 In Scope
- 指标口径统一
- 日对账任务
- 异常告警
- 重算机制

### 2.2 Out of Scope
- 自动纠偏改写历史数据（仅标记并重算）

## 3. 用户故事（US）
### US-001 产品与运营
**Description**：我希望全链路使用同一套收益口径。

**Acceptance Criteria**：
- AC-001：总资产、预估盈亏、最终盈亏口径统一。
- AC-002：跨页面同指标数值一致（允许时间窗差异标记）。

### US-002 质量保障角色
**Description**：我希望系统自动发现并报警对账异常。

**Acceptance Criteria**：
- AC-003：固定样本对账误差<=0.1%。
- AC-004：异常5分钟内触发告警。

## 4. 功能需求（FR）
- FR-001：统一指标计算库（单一来源）。
- FR-002：每日定时对账任务。
- FR-003：异常分级与告警。
- FR-004：支持按日期重算并保留版本。

## 5. 交互与流程
- 主流程：日终数据入库 → 对账任务 → 结果落库 → 告警/通过。
- 异常流程：缺数/偏差超阈值/重算失败。
- 状态流转：`pending` → `pass/warn/fail` → `recalculated`。

## 6. 接口契约（功能视角）
- 内部任务接口：对账执行、重算执行（后端任务）
- 对外查询：读取对账结果（可在后续开放）

错误码：
- FND-2002 净值数据缺失
- FND-9001 系统内部错误

## 7. 数据口径与规则
- 总资产 = Σ(份额 × 最新估值净值)
- 当日预估盈亏 = 盘中估值口径
- 当日最终盈亏 = 净值结算口径
- 累计盈亏 = 当前市值 + 已实现收益 - 累计投入
- 误差阈值：<=0.1%

## 8. 异常与降级
- 数据缺失：标记`warn`并延迟重跑。
- 任务失败：自动重试2次，失败则人工介入。
- 降级：展示最近一次通过口径版本。

## 9. 埋点与可观测
- 埋点：`reconcile_run`、`reconcile_fail`、`recalculate_run`。
- 监控：通过率、平均误差、告警时延。
- 告警：fail>0 或 warn连续2次。

## 10. 测试用例（TC）
- TC-DATA-001（AC-001）：固定样本对账通过。
- TC-DATA-002（AC-002）：跨页口径一致。
- TC-DATA-003（AC-003）：边界值场景误差可控。
- TC-DATA-004（AC-004）：异常告警时延<=5分钟。

## 11. 研发任务拆解与 DoD
### BE
- 任务：口径计算库、对账任务、重算任务。
- DoD：固定样本与边界样本通过。

### FE
- 任务：对账状态展示（如需）。
- DoD：状态可视化准确。

### QA
- 任务：样本对账、边界、跨日场景测试。
- DoD：误差阈值全部满足。

### OPS
- 任务：对账任务监控与告警链路。
- DoD：告警时延可验证。

## 12. 验收标准
- 对账误差 <= 0.1%
- 异常告警时延 <= 5分钟
- 重算任务成功率 >= 99%

## 13. 风险与待确认
- 风险：上游数据延迟引发连锁误报。
- Open Question：是否提供对账结果页面给运营。
