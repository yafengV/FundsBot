# F04 功能PRD：实时估值与首页总览

## 1. 背景与目标
- 背景问题：用户高频关注“今天赚亏多少”，需要低成本获取全局结果。
- 本功能目标：首页一屏展示核心指标，盘中分钟级刷新估值。
- 业务价值：提高日活与留存，成为产品主入口。

## 2. 范围
### 2.1 In Scope
- 首页总览卡片（总资产、当日预估、累计收益）
- 持仓列表涨跌与收益展示
- 排序筛选
- 行情延迟提示与更新时间展示

### 2.2 Out of Scope
- 高级技术指标图（MA/RSI）
- 个性化推荐排序

## 3. 用户故事（US）
### US-001 高频查看用户
**Description**：我希望打开首页就能看到核心收益，不需要多次跳转。

**Acceptance Criteria**：
- AC-001：首页展示总资产、当日预估盈亏、累计盈亏。
- AC-002：关键指标计算口径与对账结果一致。

### US-002 盘中观察用户
**Description**：我希望估值可以自动刷新，并且能知道数据是否延迟。

**Acceptance Criteria**：
- AC-003：交易时段分钟级刷新。
- AC-004：当数据延迟/失败时展示“最后更新时间+提示文案”。

## 4. 功能需求（FR）
- FR-001：`GET /dashboard/overview` 返回核心汇总指标。
- FR-002：`GET /quotes/realtime` 返回基金实时估值。
- FR-003：`GET /positions` 支持按涨跌/收益排序和筛选。
- FR-004：支持账本维度与全局维度切换。

## 5. 交互与流程
- 主流程：首页加载 → 拉取overview+positions+quotes → 展示卡片和列表。
- 异常流程：行情源超时/限流 → 展示缓存数据 + 延迟提示。
- 状态流转：`fresh`（实时）→ `stale`（延迟）→ `degraded`（降级缓存）。

## 6. 接口契约（功能视角）
- `GET /api/v1/dashboard/overview`
- `GET /api/v1/quotes/realtime?fundCodes=...`
- `GET /api/v1/positions?sortBy=...&order=...`

错误码：
- FND-2001 行情服务超时
- FND-2003 数据源配置错误（token无效）
- FND-2004 数据源积分不足
- FND-2005 数据源API限流

## 7. 数据口径与规则
- 数据源：tushare（fund_quote/fund_nav/fund_basic）。
- 口径：
  - 总资产 = Σ(份额 × 最新估值净值)
  - 当日预估盈亏 = 盘中估值口径
  - 累计盈亏 = 当前市值 + 已实现收益 - 累计投入
- 边界：非交易时段显示最近有效值，需标注时间。

## 8. 异常与降级
- 超时：请求超时后回退本地缓存。
- 重试：行情读请求最多2次指数退避。
- 降级：不可用时保留上次快照并展示“数据可能延迟”。

## 9. 埋点与可观测
- 埋点：`dashboard_view`、`overview_refresh`、`quote_delay_exposed`。
- 监控：行情成功率、刷新延迟、首页P95。
- 告警：行情成功率<99% 或刷新延迟>3分钟。
