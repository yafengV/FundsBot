# F03 功能PRD：持仓导入（CSV + OCR）

## 1. 背景与目标
- 背景问题：首次建仓需要录入大量持仓，纯手动成本高、易出错。
- 本功能目标：支持CSV与OCR导入，并通过校验与确认机制保障数据质量。
- 业务价值：提升首次导入完成率，缩短激活路径。

## 2. 范围
### 2.1 In Scope
- CSV 校验与提交
- OCR 识别草稿与人工确认
- 行级错误反馈
- 批次追踪（batchId）

### 2.2 Out of Scope
- 全自动OCR直入库（无人工确认）
- Excel复杂模板兼容（v1.1+）

## 3. 用户故事（US）
### US-001 新用户
**Description**：我希望一次性导入多个持仓，而不是逐条手动录入。

**Acceptance Criteria**：
- AC-001：CSV导入支持部分成功，并返回失败行号与原因。
- AC-002：成功记录可落库，失败记录不影响成功项。

### US-002 OCR用户
**Description**：我希望截图识别后，系统能提示不确定字段让我确认。

**Acceptance Criteria**：
- AC-003：低置信字段必须人工确认后才能提交。
- AC-004：未确认草稿不能入库。

## 4. 功能需求（FR）
- FR-001：`/imports/csv/validate` 返回校验摘要与行级错误。
- FR-002：`/imports/csv/commit` 只提交校验通过的数据。
- FR-003：`/imports/ocr/draft` 输出草稿+置信度。
- FR-004：`/imports/ocr/confirm` 仅接受已确认草稿。

## 5. 交互与流程
- 主流程：上传CSV/截图 → 校验/识别 → 用户确认 → 提交入库。
- 异常流程：字段缺失/基金代码不存在/格式错误。
- 状态流转：`uploaded` → `validated/drafted` → `confirmed` → `committed`。

## 6. 接口契约（功能视角）
- `POST /api/v1/imports/csv/validate`
- `POST /api/v1/imports/csv/commit`
- `POST /api/v1/imports/ocr/draft`
- `POST /api/v1/imports/ocr/confirm`

错误码：
- FND-1001 基金代码不存在
- FND-1002 参数缺失/格式错误
- FND-1003 业务校验失败

## 7. 数据口径与规则
- 导入成功后写入：`position` + `position_txn`。
- 校验规则：基金代码、份额、成本、日期格式。
- 口径：部分成功可用，不回滚全批次。

## 8. 异常与降级
- OCR服务超时：返回“稍后重试”，保留已上传文件。
- 提交失败：返回批次失败原因，可重复提交通过项。
- 幂等：同批次校验码重复提交需去重。

## 9. 埋点与可观测
- 埋点：`import_csv_upload`、`import_csv_commit`、`import_ocr_confirm`。
- 监控：导入成功率、字段错误率、OCR确认率。
- 告警：导入失败率>10%（10分钟窗口）。

## 10. 测试用例（TC）
- TC-IMP-001（AC-001）：CSV全量成功。
- TC-IMP-002（AC-001）：CSV部分失败返回行号。
- TC-IMP-003（AC-003）：OCR低置信字段标红。
- TC-IMP-004（AC-004）：未确认草稿不可提交。

## 11. 研发任务拆解与 DoD
### BE
- 任务：CSV校验/提交、OCR草稿确认、批次追踪。
- DoD：行级错误可追踪，部分成功逻辑通过。

### FE
- 任务：上传页、校验结果页、OCR确认页。
- DoD：错误可定位到行/字段。

### QA
- 任务：全成功/部分成功/异常数据测试。
- DoD：AC映射用例100%执行。

### OPS
- 任务：导入失败监控、OCR超时告警。
- DoD：告警链路可验证。

## 12. 验收标准
- 导入完成率 >= 85%
- 导入错误可定位率 100%
- OCR低置信拦截准确率 >= 99%

## 13. 风险与待确认
- 风险：OCR质量波动影响转化率。
- Open Question：批次历史是否在v1.0可视化展示。
