# F02 功能PRD：持仓管理（手动增删改 + 增减仓）

## 1. 功能目标
实现基金持仓全生命周期管理，支持新增、编辑、增仓、减仓、清仓。

## 2. 用户故事
- 作为用户，我希望准确维护每只基金的份额与成本，看到真实收益。

## 3. 功能范围
- 新增持仓
- 编辑持仓备注/平台
- 增仓、减仓
- 清仓状态管理
- 删除（软删）

## 4. 业务规则
1. 基金代码必须存在于基金主数据
2. 份额必须 > 0（新增/增仓）
3. 减仓份额不得超过持有份额
4. 清仓后状态为 `cleared`，历史记录保留

## 5. 核心计算规则
- 增仓后加权成本：
  `new_cost = (old_shares*old_cost + add_shares*add_nav)/(old_shares+add_shares)`

## 6. 接口映射
- `POST /api/v1/positions`
- `PATCH /api/v1/positions/{id}`
- `DELETE /api/v1/positions/{id}`
- `GET /api/v1/positions`

## 7. 数据结构映射
- 表：`position`, `position_txn`

## 8. 测试用例（按功能）
- F02-TC-001 新增持仓成功
- F02-TC-002 基金代码不存在失败
- F02-TC-003 增仓后成本正确
- F02-TC-004 超额减仓失败
- F02-TC-005 清仓状态正确

## 9. 验收标准
- 持仓计算准确率 >= 99.9%
- 核心路径（新增/增减仓）通过率 100%
