# 开发落地说明（DDL + OpenAPI + Seed）

本次已补齐可直接进入研发的三项产物：

1. **数据库结构（DDL）**
   - 文件：`db/schema.sql`
   - 范围：用户、账本、基金主数据、持仓、交易记录、实时估值、日净值、日报快照、报表快照、提醒规则、幂等键

2. **接口定义（OpenAPI 3.0）**
   - 文件：`api/openapi/fundsbot.v1.yaml`
   - 范围：账本、持仓、导入、总览、提醒、报表
   - 可用于：mock server、前后端 contract test、自动生成 SDK

3. **测试/联调种子数据**
   - 文件：`db/seed.sql`
   - 含：1个用户、2个账本、3只基金、3条持仓、净值和提醒规则

## 建议执行顺序
1. 后端先落库 `schema.sql`
2. 导入 `seed.sql` 做本地联调
3. 前后端按 `fundsbot.v1.yaml` 联调
4. QA 基于 `docs/prd/07-测试用例与验收矩阵.md` 跑首轮测试
